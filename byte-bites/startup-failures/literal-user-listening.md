# 🎓 Literal User Listening vs. Data Literacy

Building exactly what users ask for can fail if users do not actually understand the underlying technical requirements of the problem.

---

## Lago v1 (YC S21)
* **What they built**: A no-code "Reverse ETL" tool allowing marketers to sync data from company databases to tools without writing SQL.
* **The Failure**: Marketers said they wanted a no-code UI to avoid SQL. The founders built it, but realized marketers still couldn't use it because they lacked basic "data literacy" (table relations, database modeling).
* **The Outcome**: Rebuilt Lago v2 as an open-source developer-first metering and billing API, which achieved massive product-market fit.

---

### 💡 Key Takeaway
For startups in this category, the core challenge is not the code but the surrounding market dynamics. Ensure you validate this bottleneck before scaling.
